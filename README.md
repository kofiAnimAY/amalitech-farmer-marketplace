# Amalitech Farmer Marketplace

## Overview

This project implements a simple Flask-based farmer-to-market produce marketplace API using Flask-RESTX and JWT authentication.

Supported endpoints:
- `POST /register/register` — register a new user
- `POST /register/login` — authenticate and receive a JWT
- `GET /marketplace/listings` — retrieve all listings
- `POST /marketplace/listings` — create a new listing (farmers only)
- `PATCH /marketplace/listings` — update a listing (owning farmer only)
- `DELETE /marketplace/listings` — delete a listing (owning farmer only)
- `POST /marketplace/buy` — buy from a listing (buyers only)

## Setup

1. Create a Python virtual environment and install dependencies:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

2. Configure required environment variables.

Create a `.env` file or set the variables in your environment:

```env
JWT_KEY=your-secret-key
MONGO_URI=mongodb://localhost:27017/testdb
DB_NAME=testdb
MOCK_DB=true
```

3. Run the application:

```powershell
python app.py
```

The API runs at `http://127.0.0.1:5000` by default.

## Authentication and Authorization

- The API uses JWT Bearer tokens for protected endpoints.
- Users register with a `role` of either `farmer` or `buyer`.
- Tokens are generated by `POST /register/login`.
- Protected endpoints require the header:

```http
Authorization: Bearer <token>
```

### Roles

- `farmer`: can create listings, update own listings, and delete own listings.
- `buyer`: can purchase from listings.

## Endpoints

### Register a new user

`POST /register/register`

Request body:

```json
{
  "username": "alice",
  "email": "alice@example.com",
  "password": "password123",
  "role": "farmer"
}
```

Response:

- `201 Created` with `user_id` on success
- `409 Conflict` if email already exists

### Login

`POST /register/login`

Request body:

```json
{
  "email": "alice@example.com",
  "password": "password123"
}
```

Response:

- `200 OK` with `token`
- `401 Unauthorized` for invalid credentials

### Get all listings

`GET /marketplace/listings`

This endpoint is public and returns all listings.

Response:

- `200 OK`

Example:

```json
{
  "listings": [
    {
      "_id": "...",
      "item": "tomato",
      "description": "fresh",
      "price": 1.23,
      "quantity": 10,
      "listed_by": "..."
    }
  ]
}
```

### Create a new listing

`POST /marketplace/listings`

Requires:
- authenticated user with role `farmer`

Request body:

```json
{
  "item": "tomato",
  "description": "fresh",
  "price": 1.23,
  "quantity": 10
}
```

Response:

- `201 Created` on success
- `400 Bad Request` if required fields are missing
- `401 Unauthorized` if token missing or invalid
- `403 Forbidden` if user is not a farmer

### Update a listing

`PATCH /marketplace/listings`

Requires:
- authenticated user with role `farmer`
- owner of the target listing

Request body:

```json
{
  "listing_id": "<listing_id>",
  "new_price": 2.50,
  "new_quantity": 20,
  "new_name": "cherry tomato",
  "description": "organic"
}
```

Response:

- `200 OK` on success
- `400 Bad Request` if `listing_id` is missing
- `403 Forbidden` if the authenticated farmer does not own the listing
- `404 Not Found` if the listing does not exist

### Delete a listing

`DELETE /marketplace/listings`

Requires:
- authenticated user with role `farmer`
- owner of the target listing

Request body:

```json
{
  "listing_id": "<listing_id>"
}
```

Response:

- `200 OK` on success
- `400 Bad Request` if `listing_id` is missing
- `403 Forbidden` if the authenticated farmer does not own the listing
- `404 Not Found` if the listing does not exist

### Buy from a listing

`POST /marketplace/buy`

Requires:
- authenticated user with role `buyer`

Request body:

```json
{
  "listing_id": "<listing_id>",
  "quantity": 2
}
```

Response:

- `200 OK` on success with purchase cost message
- `400 Bad Request` if required fields are missing
- `403 Forbidden` if user is not a buyer
- `404 Not Found` if the listing is not found or quantity is insufficient

## Notes

- Only the farmer who created a listing may update or delete it.
- `GET /marketplace/listings` is publicly accessible and does not require authentication.
- All protected endpoints require the `Authorization` header.
- Ensure `JWT_KEY` is set consistently for both login and protected endpoint validation.
